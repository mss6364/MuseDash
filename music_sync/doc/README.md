# Music Sync（软硬件协同）

目标：启动后监听键盘，按空格播放对应音乐；若无同名音乐则可播放节拍。

实现要求（当前仅保留接口占位，需按此补全）：
- 核心接口：`listen_and_play(chart_path_or_audio_path)`
  - 监听键盘，按下空格立即播放。
  - 输入可为曲目名或谱面/音频路径；优先查找 `charts/<曲目名>/<曲目名>.mp3`，无同名音频则回退播放节拍。
- 调试入口：`main` 仅用于手动测试。

软硬件协同探索（可由 AI 给出方案，无需实现）：
- 思路：通过键盘事件/串口/USB HID 将按键状态传给 FPGA，控制播放开关或节拍输出。
- 可选方案：
  1) 在 PC 侧捕获键盘事件，经串口发送简单协议给 FPGA（如 `SPACE_DOWN` / `SPACE_UP`）。
  2) 使用 USB HID 仿真键盘/自定义设备，由 FPGA 读取状态驱动播放。
  3) 借助 AI 生成简易协议与 Verilog 示例，描述状态机如何解析指令控制音频模块或节拍器。
  仅需思路，无需实际落地。

目录说明：
- `player.py`：仅保留接口占位，与上述要求保持一致。
- `assets/`：默认音乐/节拍资源存放处（可选）。
